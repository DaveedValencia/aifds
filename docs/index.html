AIFDS Documentation

The AI Feed Discovery Standard defines lightweight JSON layouts so agents can understand your business without crawling every page. Start here to explore the file anatomy and roadmap.

Schema Family

  • Ecommerce: homepage.schema.json, collections.schema.json, pdp.schema.json
  • Content: blog.schema.json, pages.schema.json
  • Future Industries: Services, SaaS, Media (coming soon)

Each schema pairs with a matching example JSON document under /examples. Both files will evolve together as the standard matures.

Design Principles

  • Snake_case keys with predictable nesting so agents can traverse the object tree deterministically.
  • Time-stamped documents via version and updated_at to express freshness.
  • Action-forward data: highlight what a user or agent can do (search, checkout, contact) instead of prose.
  • Evidence slots for credibility signals (directories, reviews, ownership) baked into every layout.

What to Expect Next

  • Validation harness that reads AIFDS JSON and scores readiness for LLM consumption.
  • Industry playbooks showing how to adapt the base schema to services, SaaS, and media properties.
  • Guides for publishing AIFDS JSON alongside traditional site content without leaking private data.
