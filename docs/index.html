How to work with AIFDS

Keep these guidelines handy as you adapt the schemas to your own stack.

Layout rhythm

Each template follows a consistent order: context → evidence → actions. Maintain this sequence so downstream agents always know where to look.

Versioning

Update the `version` and `updated_at` fields every time the JSON changes. Agents rely on those timestamps to decide when to refresh cached knowledge.

Publishing

Host JSON at stable, crawlable URLs. Reference them from `robots.txt`, site maps, and the `Link` HTTP header when possible.
