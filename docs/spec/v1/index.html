AI Schema Discovery Standard (AIFDS) v1 — Draft

Draft guidance for publishing schema.org-aligned JSON payloads that keep commerce and service data AI-ready.

Tracks in v1

  • Commerce Schema Family (Shopify-first) — finalized drafts for product.json, homepage.json, collections.json, and blog.json.
  • Service Schema Family — draft schemas for service-homepage.json, services.json, contact.json, and pricing.json.

Commerce Schema Family

The commerce track translates core Shopify surfaces into schema.org-aligned JSON. Implementers can adopt the drafts below to keep merchandising state synchronized across AI partners.

product.json

Maps a shoppable SKU to schema:Product, enforcing canonical identifiers, merchandising facts, and offer integrity.

  • Required payload keys: @context, @type, sku, name, description, url, offers, lastModified.
  • offers must include price, currency, availability, and PDP URL. Use schema.org URLs for availability enums.
  • Variant relationships are captured via hasVariant referencing sibling SKU identifiers.

homepage.json

Captures the storefront landing page as schema:WebPage with structured hero sections, featured collections, and conversion CTAs.

  • Use hasPart to describe sections such as hero banners, promo carousels, or offer catalogs.
  • Each section may reference downstream collections or products via itemListElement.
  • Surface brand identity through mainEntity with links to organization or brand JSON-LD.

collections.json

An array of schema:CollectionPage objects with embedded schema:ItemList entries describing merchandising groups.

  • Reference products using @id or sku to align with product.json.
  • Document shopper experience with available filters, sort options, and SEO metadata.
  • Track freshness through lastModified timestamps per collection.

blog.json

Models the editorial layer as schema:Blog with a curated list of schema:BlogPosting entries.

  • Every post must carry canonical URLs, publication dates, author metadata, and summary content.
  • Use mentions to cross-link products or services highlighted in the article.
  • Voice-ready excerpts can be described with speakable markup.

Service Schema Family

The service track documents how agencies and studios articulate offers, proof, and pricing to AI distribution partners.

service-homepage.json

Models the primary services landing page as schema:WebPage with semantic sections for hero messaging, proof, and CTAs.

  • Use sectionType labels (hero, services, testimonials, etc.) to keep downstream experiences consistent.
  • Testimonials are expressed via schema:Review objects to retain author, publisher, and quote metadata.

services.json

An array of schema:Service entries capturing deliverables, audiences, operating hours, and offer metadata.

  • Every service must publish @id, serviceType, and lastModified for traceability.
  • Cross-link pricing packages with hasOfferCatalog referencing the pricing.json catalog.
  • Use deliverable and workExample arrays to surface proof artifacts.

contact.json

Defines intake expectations via schema:ContactPoint including languages, SLAs, and workflow steps.

  • Normalize availability with schema:OpeningHoursSpecification objects.
  • Express response commitments using ISO 8601 durations in the serviceLevelAgreement block.

pricing.json

Captures offer bundles as a schema:OfferCatalog with ordered packages mapped back to services.

  • Each package should reference a service via itemOffered.@id and expose price specifications or negotiability.
  • Include advanceBookingRequirement and eligibleCustomerType to set buyer expectations.

Validation Workflow

  1. Generate payloads from source systems (Shopify, CMS, service catalogs) into the JSON schemas published at /schemas/v1/.
  2. Validate with JSON Schema Draft 2020-12 compatible tooling before distribution.
  3. Publish validation reports alongside feeds so partners can diff payload changes (lastModified fields) and assess impact quickly.
