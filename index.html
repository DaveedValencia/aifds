Why agents need AIFDS

  • Self-contained facts: ownership, operations, actions, and credibility in one file.
  • Predictable structure that lets LLMs map questions to answers without crawling.
  • Versioned JSON so downstream systems trust freshness signals instantly.

Current focus: Ecommerce

  • Homepage JSON template (`site → about → credibility → contact → operations → navigation → highlights → actions`).
  • Collections, PDP, and marketing pages staged as placeholders ready for field definitions.
  • Validator design underway to grade readiness (understands, answers, acts).

Industry roadmap

We are sequencing page types by evidence needs. Ecommerce ships first; Services, SaaS, and Media follow using the same modular blocks.

  • Ecommerce: homepage, collections, product detail, support content.
  • Services: credentialed staff, coverage zones, booking actions.
  • SaaS: pricing tiers, integrations, success metrics.
  • Media: catalog navigation, licensing, distribution partners.

How to build with AIFDS

  • Create the matching schema + example pair under `/schemas` and `/examples`.
  • Publish the JSON next to your site so agents can fetch it via HTTPS.
  • Track `version` and `updated_at` to broadcast change history.
  • Map every call-to-action (`search`, `checkout`, `contact`) to a resolvable URL.
